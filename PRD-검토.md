# PsychPaper PRD 검토 요약

> PRD v1.0 (2026-02-11) 검토 결과입니다. 쉽게 말해 "이 문서가 잘 쓰였는지, 빠진 건 없는지" 정리한 거예요.

---

## 잘된 점 (강점)

| 항목 | 이유 |
|------|------|
| **문제 정의가 분명함** | P1~P5로 "누가 왜 불편한지"가 표로 정리되어 있어서, 나중에 "이 기능 왜 만드나?" 할 때 PRD만 보면 됨 |
| **PARS 체계** | 현상/응용/연구유형/규모 4축 분류가 구체적이고 예시가 많아서, DB 설계·UI(탭/뱃지) 설계 시 그대로 쓸 수 있음 |
| **MVP 범위 잘라냄** | 독자 로그인, 좋아요, 뉴스레터, AI 요약 등은 "제외"로 명시해서, 첫 버전에 꼭 필요한 것만 하기 좋음 |
| **데이터 구조·API 정의** | articles, categories, comments 등 테이블/엔드포인트가 나와 있어서 백엔드 설계 시 바로 참고 가능 |
| **디자인 가이드라인** | 컬러 변수, 폰트, 다크 테마 원칙이 있어서 기존 Deep Dive HTML과 통일감 유지하기 좋음 |
| **완료 기준 체크리스트** | MVP 끝났는지 확인할 때 사용할 수 있는 체크 항목이 있어서 좋음 |

---

## 보완하면 좋은 점

| 항목 | 제안 |
|------|------|
| **slug 유일성** | 아티클 `slug`가 URL에 쓰이므로, DB에서 unique 제약 + "이미 있으면 slug 자동 변경" 같은 규칙을 PRD나 DB 스키마에 한 줄 넣으면 좋음 |
| **댓글 비밀번호** | "비밀번호(선택)"인데, 수정/삭제 시 이 비밀번호로 본인 확인할지 여부를 PRD에 명시해 두면 구현 시 헷갈리지 않음 |
| **이미지 업로드** | "대표 이미지 업로드 또는 자동 생성" — MVP에서 자동 생성은 제외됐으니, "MVP에서는 업로드만"이라고 한 줄 적어 두면 명확함 |
| **Scale(규모) 저장 방식** | S는 라벨(소규모/중규모/대규모/메가) + 참가자 수인데, DB에 "scale_label" + "participant_count" 둘 다 넣을지, 라벨만 넣을지 정해 두면 좋음 |
| **에러/빈 상태** | "카테고리에 아티클이 0개일 때", "검색 결과 없을 때" 같은 빈 상태 UI를 PRD 7장에 한 줄씩 넣어 두면 디자인 일관성에 도움됨 |

---

## 기술적으로 확인할 부분

| 항목 | 내용 |
|------|------|
| **Deep Dive HTML 보안** | HTML 본문을 그대로 렌더링할 경우 XSS 위험이 있음. "신뢰할 수 있는 작성자만 HTML 업로드" 전제이더라도, sanitize(DOMPurify 등) 적용하는 것을 권장하고, PRD 9장 또는 보안 규칙에 적어 두면 좋음 |
| **OG 메타태그** | SPA만 쓰면 SNS 크롤러가 og:image 등을 못 읽을 수 있음. "SSR 또는 프리렌더링"이라고 PRD에 이미 적혀 있어서, Next.js로 전환하거나 별도 OG용 라우트를 둘 계획이면 그걸 기술 스택에 명시해 두면 좋음 |
| **조회수** | "조회수 중복 방지"는 MVP 제외됐지만, 같은 사용자가 10초 안에 여러 번 새로고침하면 +1만 되게 하는 정도의 간단 규칙은 Phase 1에서 넣어도 부담 적음 |

---

## 규칙 적용 시 반영한 내용

- **프로젝트 규칙(.cursor/rules)** 에 PRD의 다음 내용을 반영했습니다.
  - 프로젝트 개요: PsychPaper 목적, 1차/2차 사용자, PARS 정의
  - 아키텍처: React + Express + PostgreSQL, 폴더 구조
  - DB: articles, categories, comments 등 스키마 요약
  - API: Public / Admin 엔드포인트
  - UI: 다크 테마, 컬러, 폰트, 컴포넌트 목록
  - 기술 규칙: 네이밍, TypeScript, 에러 처리
  - 보안: JWT, 댓글 비밀번호, HTML sanitize 권장
  - 테스트/배포: Jest, Docker, Phase별 우선순위

디자인 파일을 주시면, 이 규칙들과 PRD 11장(디자인 가이드라인)에 맞춰 컴포넌트/스타일 가이드를 더 구체화할 수 있습니다.
